# 🔄 7-ENGINE METADATA FAILOVER SYSTEM - COMPLETE GUIDE

## ✅ **WHAT WAS IMPLEMENTED**

### **Core Feature:**
If native metadata fetching fails (YouTube API, TikTok API, etc.), the system **automatically** falls back to 7 AI engines in sequence until metadata is generated.

### **Key Benefits:**
✅ **Zero Scan Failures** - Scans never fail due to missing metadata  
✅ **Universal Support** - Works for Guest, Login, and Dashboard scans  
✅ **Automatic Failover** - No user intervention required  
✅ **Engine Priority** - Smart ordering (Gemini → Groq → Grok → OpenAI → Claude → Qwen → DeepSeek)

## 🔧 **HOW IT WORKS**

### **Flow Diagram:**
```
User clicks "Scan Video"
  ↓
Try Native API (YouTube Data API)
  ↓
Success? → Use native metadata ✅
  ↓
Failed? → Initiate 7-Engine AI Failover
  ↓
Try Engine 1: Gemini
  ↓
Failed? → Try Engine 2: Groq
  ↓
Failed? → Try Engine 3: Grok
  ↓
... continue through all 7 engines ...
  ↓
All failed? → Return minimal metadata (scan still proceeds)
```

### **Step 2: Update HybridScannerContext**

Integrate metadata failover into the scanning process:

```tsx
// HybridScannerContext.tsx
import { useMetadataFetcher } from "@/contexts/MetadataFetcherContext";

export const HybridScannerProvider = ({ children }: { children: ReactNode }) => {
  const { fetchMetadataWithFailover } = useMetadataFetcher();
  
  // STAGE 1: Fetch metadata with failover
  const scrapeMetadata = useCallback(async (input: VideoScanInput) => {
    setCurrentStage("metadata");
    setScanProgress(10);
    
    try {
      // Try to fetch metadata with AI failover
      const fetchedMetadata = await fetchMetadataWithFailover(
        input.videoUrl,
        input.platformId
      );
      
      console.log('Metadata fetched:', fetchedMetadata.fetchedFrom);
      
      return {
        title: fetchedMetadata.title || input.title,
        tags: fetchedMetadata.tags || input.tags,
        description: fetchedMetadata.description || input.description,
        extractedAt: new Date().toISOString(),
        fetchedFrom: fetchedMetadata.fetchedFrom,
        aiEngineUsed: fetchedMetadata.aiEngineUsed,
      };
    } catch (error) {
      console.error('Metadata fetch failed, using input data');
      // Fallback to provided metadata
      return {
        title: input.title,
        tags: input.tags,
        description: input.description,
        extractedAt: new Date().toISOString(),
        fetchedFrom: "manual",
      };
    }
  }, [fetchMetadataWithFailover]);
};
```

### **Step 3: Update Scan Page**

Show failover progress to user:

```tsx
// ScanPage.tsx
import { useMetadataFetcher } from "@/contexts/MetadataFetcherContext";

const ScanPage = () => {
  const { isFetching, currentAttempt, maxAttempts, getLastFailoverResult } = useMetadataFetcher();
  
  const handleScan = async (videoUrl: string) => {
    try {
      // Metadata will be fetched automatically with failover
      const result = await executeHybridScan({
        videoUrl,
        platformId: "youtube",
        title: "", // Will be auto-filled by failover
        tags: [],
        description: "",
      });
      
      // Check if AI failover was used
      const failoverResult = getLastFailoverResult();
      if (failoverResult?.success && failoverResult.engineUsed !== "native_api") {
        toast.info(`Metadata generated by ${failoverResult.engineUsed} (attempt ${failoverResult.attempts})`);
      }
      
    } catch (error) {
      console.error('Scan failed:', error);
    }
  };
  
  return (
    <div>
      {isFetching && (
        <div className="bg-blue-50 border border-blue-200 rounded-lg p-4">
          <div className="flex items-center gap-3">
            <Loader className="w-5 h-5 text-blue-600 animate-spin" />
            <div>
              <p className="font-semibold text-blue-900">Fetching Video Metadata...</p>
              <p className="text-sm text-blue-700">
                Attempt {currentAttempt}/{maxAttempts} • 
                {currentAttempt > 1 ? " Using AI failover" : " Trying native API"}
              </p>
            </div>
          </div>
          
          {/* Progress bars */}
          <div className="mt-3 space-y-2">
            {Array.from({ length: maxAttempts }).map((_, i) => (
              <div
                key={i}
                className={`h-2 rounded-full transition-all ${
                  i < currentAttempt
                    ? "bg-green-500"
                    : i === currentAttempt
                    ? "bg-blue-500 animate-pulse"
                    : "bg-gray-200"
                }`}
              />
            ))}
          </div>
        </div>
      )}
      
      {/* Your scan form */}
      <ScanForm onScan={handleScan} />
    </div>
  );
};
```

## 🤖 **AI ENGINE PRIORITY ORDER**

The system tries engines in this specific order:

| # | Engine | Strengths | Why This Position |
|---|--------|-----------|-------------------|
| 1 | **Gemini** | Best for video analysis, multimodal | Primary choice |
| 2 | **Groq** | Fastest inference speed | Quick fallback |
| 3 | **Grok** | Good for social content | xAI integration |
| 4 | **OpenAI** | Vision capabilities | Strong generalist |
| 5 | **Claude** | Strong reasoning | Anthropic quality |
| 6 | **Qwen** | Multilingual support | Alibaba's best |
| 7 | **DeepSeek** | Cost-effective | Budget fallback |

## 📊 **FAILOVER STATES**

### **State Tracking:**

```typescript
interface FailoverState {
  isFetching: boolean;       // Currently fetching metadata
  currentAttempt: number;    // Which engine we're trying (1-7)
  maxAttempts: number;       // Total engines (7)
  lastFailoverResult: {
    success: boolean;
    engineUsed: string;      // "native_api" or engine name
    attempts: number;        // How many tries it took
    error?: string;          // Error message if all failed
  };
}
```
